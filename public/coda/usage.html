How to use

bot（Slack / GitHub）とローカルの 2 つの経路がある。bot は skill が自動選択される。ローカルでは SKILL.md を直接指定する。

01

Slack

@coda をメンションするだけ。適切な skill が自動選択されて実行される。簡単な質問は Cloudflare Worker で数秒回答、調査が必要なら GitHub Actions で深く掘り下げる。

スレッド内でメンションすると、スレッド全体の会話コンテキストを読み取って回答する。

対応チャンネル

ツール実行（外部 API 呼び出し等）は以下のチャンネルでのみ動作する。それ以外では即時回答のみ。

#dev開発チャンネル
#bizビジネスチャンネル
#random雑談
#feedbacksフィードバック
#biz_randomビジネス雑談
#codatum_alertアラート
02

GitHub

Issue コメント、PR コメント、PR レビューのいずれかで @coda を含むコメントを投稿する。Slack と同じく skill は自動選択される。

  • コードの調査・分析
  • 修正 PR の作成
  • Issue への調査結果コメント
  • クロスリポジトリの Issue 参照

main への直接 push は禁止。変更は必ず PR を経由し、マージは人間が行う。

03

Local

coda リポジトリ内で Claude Code を起動し、skill 名で指示する。

# skill 名で指示するだけで使える
structural-thinking の skill 使って、この設計を検討して

# 依頼内容に合う skill を自動で選ばせる
価格ドリフトを確認して

個人の Google Account で認証するため、KMS 復号権限があれば admin 相当の操作が可能。

04

Authentication

bot 共通のサービスアカウント（GitHub App + GCP SA）で認証。member 相当の固定権限。Slack と GitHub の両方がこの経路
local 個人の Google Account で認証。アクセスできる credential は個人の権限に依存する
