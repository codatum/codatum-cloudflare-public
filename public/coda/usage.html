About Skills Usage Security

How to use

coda の使い方。ローカル、Slack、GitHub の 3 つの経路がある。

01

Authentication

経路によって認証が異なる。

Local 個人の Google Account で認証。SOPS + GCP KMS の復号権限が必要。アクセスできる credential は個人の権限に依存する。
Slack / GitHub 共通のサービスアカウント（GitHub App + GCP SA）で認証。GitHub Actions 上で動作し、member 相当の固定権限。両方とも同じ credential セットを使用する。
02

Local

セットアップ

# 1. clone
git clone git@github.com:codatum/coda.git

# 2. 依存関係・credential の復号
make setup

# 3. 対象リポジトリから symlink
mkdir -p .claude
ln -s /path/to/coda .claude/skills

make setup には GCP KMS の復号権限（roles/cloudkms.cryptKeyDecrypter）が必要。

skill を直接指定して実行

# 対話モード
make claude-skill SKILL=workflows/llm-pricing-workflow

# 非対話モード
make claude-skill SKILL=workflows/llm-pricing-workflow MODE=once

あいまいな依頼をルーティング

make claude-route TASK="価格ドリフトとmissingを確認して"

Claude Code から直接読み込む

symlink を設定済みなら、Claude Code が .claude/skills/ 配下の SKILL.md を自動で認識する。SKILLS.md を読ませて適切な skill を選ばせることもできる。

03

Slack

基本

@coda をメンションする。簡単な質問には Cloudflare Worker で数秒回答し、調査が必要な場合は GitHub Actions で深く掘り下げる。

スレッド対応

スレッド内でメンションすると、スレッド全体の会話コンテキストを読み取って回答する。

対応チャンネル

ツール実行（外部 API 呼び出し等）は以下のチャンネルでのみ動作する。それ以外では即時回答のみ。

#dev開発チャンネル
#bizビジネスチャンネル
#random雑談
#feedbacksフィードバック
#biz_randomビジネス雑談
#codatum_alertアラート
04

GitHub

トリガー

Issue コメント、PR コメント、PR レビューのいずれかで @coda を含むコメントを投稿する。

できること

  • コードの調査・分析
  • 修正 PR の作成
  • Issue への調査結果コメント
  • クロスリポジトリの Issue 参照

注意

main への直接 push は禁止。必ず PR を作成し、マージは人間が行う。
